Baldr separates two cycles: **designing and publishing** an agent, and
**resolving and operating** that release. The first belongs to the team that
creates the agent; the second is the platform's responsibility.

<ArchitectureExplorer />

## The two paths

### Coordinate a task

```text
surface -> Router -> Agent Manager -> Runner -> workspace -> evidence
```

Router pins the workflow and team. Agent Manager returns exact identities.
Runner verifies and executes. The result returns to durable state and is
projected to the surface without exposing reasoning or private logs.

### Publish an agent

```text
repository -> SDK -> Agent Builder -> driver -> immutable release
```

The checkout is never copied into Router. Builder produces a portable artifact
and one manifest per role. Publishing installs a stable version that can later
be activated, audited, or rolled back.

## Three planes

| Plane | Contains | Must not contain |
| --- | --- | --- |
| Control | workflows, policies, identity, and decisions | private agent code |
| Data | Runner processes, limits, and results | product selection |
| Artifacts | releases, manifests, and evidence | mutable checkout references |
